This is a common (and basic) problem in 3d graphics. I forgot the exact terminology, but solving this involves translating the 2d coordinates of the mouse to where it ACTUALLY is referring to in 3d space.
 
One way is to keep a "dummy" object oriented to the camera, but have the mouse control the dummy object's X and Y, in relation to the camera/viewing plane. Then draw a line from this dummy object to infinity (or.. a very long length). This line is now perpendicular to the camera plane, and thus, the screen. Whichever object intersects with this line is what the user is actually trying to hit. If there are two, you might need a routine to sort things out by z-depth.
 
I hope that helps a bit.
