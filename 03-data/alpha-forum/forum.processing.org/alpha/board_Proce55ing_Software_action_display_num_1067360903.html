Re: jar files exported-to-web separately
« Reply #3 on: Jul 7th, 2004, 5:37pm »
so casey and i are discussing this one a bit, but haven't been able to make a decision on the best route.  
 
pros for single jar:  
1) simple sketches won't need to have multiple files, or a jar for a single .class file.  
2) exporting to a double-clickable jar file will require a single .jar anyway, so to support alternate methods of packaging for different types of export can introduce new problems/bugs/annoyances
3) are there really that many instances where you have many p5 applets in a single folder?
 
in favor of multiple jar:
1) why break up perfectly good .jar files and potentially cause problems when they're merged back together?
2) speed increase for people with many applets in one folder.  
 
anyone care to share some input on this one?
