task description
platform maintainer for your favorite os.
 
minimum commitment
a few hours, once a month. i'll send an email saying that the current version in cvs is ready to go for the next release. you get the latest rev, build it, and test it (run through the examples and try basic coding). if all is well, you package the release using the distribution script, and post it online where it can be added to the site. ideally we'd like you to be able to get this done within 24-48 hours of getting the email. usually, it can be done quickly (minimum of 30 minutes, max of a few hours if there are bumps)
 
further commitment
if interested, you can take care of more of the tasks related to your specific platform. i.e. on macos, making sure that the distribution is as 'mac like' as possible (.dmg files with pretty icons).
or on linux, making sure that the jvm that's being used will run on most platforms (redhat 7/8/9 in particular), or providing additional install instructions for upgrading the jvm.
 
if you're unable to do the 'further' commitment but might be able to do the first part, then we'll break that other stuff out as a separate job, since that stuff is still important, and useful finishing details for a nice product.
 
hey! that's ME!
if interested, send email to processing@media.mit.edu with details of your interest level and some background on yourself.  
 
questions?
if you have general questions, *please please please* respond here, so that others can benefit from your query (most likely, someone else will have the same question, so posting here saves us time)
