Re: proce55inf fails at startup on OSX (jaguar)
« Reply #12 on: Dec 7th, 2002, 12:26am »
it's freaking out about the .DS_Store file being created in the sketchbook directory.  
 
for the time being, if you run into the problem, just remove the .DS_Store file in your sketchbook directory.
 
that fixed the crashing with the version that was posted for me, not sure if other .DS_Store files elsewhere in the sketchbook cause the same trouble, but this would explain the strange intermittency of the bug--that things work for a while, until you open the folder in the finder and the DS_Store file gets created.
 
i think DS_Store is being reported as a folder, rather than a file, by java. at any rate, i'll have a workaround for the problem in 47.
 
thanks everyone for the thorough reporting..
