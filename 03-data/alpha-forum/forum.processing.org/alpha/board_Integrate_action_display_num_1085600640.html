Re: Processing "headless" mode?

The problem is that if you want to run a Java server app it might be running without access to the display, and hence fail with an exception when the app tries to create a window.
 
Java apps can invoke a headless mode in which the AWT functions are called without windows. See more information here. That documents seems to indicate that java.awt.Applet (which BApplet extends) will throw a HeadlessException.
 
I guess the best way to find out definitively is to try it.

Re: Processing "headless" mode?
« Reply #6 on: Sep 4th, 2004, 7:14pm »
one option for headless might be using just BGraphics and not BApplet. this would mean you'd get the graphics stuff but lose a little of the math (sin/cos/etc) and things like time and input events (no big deal, these are all wrappers on java functions.  
 
re: what happens when 'run' is hit.. the code for your .pde file (what's showing in the p5 window) gets run through a preprocessor that converts the code to straight java. the code is then compiled using jikes, and then loaded in one of two ways. if you're not using a code folder, Class.forName() is used to create an instance of your class (the class files that were built by jikes are in lib/build), which will have a fake name on it (Temporary_NNNN_NNNN). for things with a code folder, a new java vm is started that runs your code via "java BApplet <yoursketchname>" using the compiled classes from jikes.
