just tacked in a fix for this for rev 64, which will be online in about an hour.  
 
the problem was that safari sends &nbsp; as a non-breaking space char, which is a special unicode character that looks the same but causes headaches for programs.  
 
i added a little code that replaces the nbsp with a regular space whenever you paste code into the environment.
